Run simulation timestep

Usage

run_timestep(metacomm, land, species, gsad, d_kernel = NULL, v_kernel = NULL, imm_rate = NA, return_dead = F)

Arguments

metacomm
(required) matrix of lists defining metacommunity (as generated by populate_landscape or returned by this function).
land
(required) matrix or raster of habitat types defining the landscape (as generated by make_landscape)
species
(required) array of species vital rates (as generated by make_species)
gsad
(required) vector defining the global relative abundance of each species. Must be in the same order as species. Defaults to same abundance for all species.
d_kernel
list defining the dispersal kernel of new propagules (see get_dispersal_vec for options). Passed to the form parameter in disperse. Defaults to Gaussian.
v_kernel
list defining the dispersal kernel of previously established individuals (see get_dispersal_vec for options). Passed to the form parameter in disperse. Defaults to Gaussian.
imm_rate
immigration rate. Passed to the parameter m in establish. Defaults to 0.
return_dead
logical indicating whether a second metacommunity array should be returned that shows which individuals dies during this timestep. Used for calculating mortality and establishement rates. Defaults to FALSE.

Value

if return_dead is FALSE, then the function returns a matrix of lists defining the metacommunity with the same dimensions as metacomm, otherwise it returns a list containing two metacommunity arrays- the first is the new metacommunity and the second is an array of logicals indicating whether each microsite was vacated (through death or movement) in this timestep.

Description

Runs the simulation on a metacommunity for one timestep.

Details

Given a metacommunity (metacomm, see populate_landscape), landscape (land, see make_landscape), species pool (species, see make_species), and global species abundance distribution (gsad), this function calls the main process functions (die, reproduce, disperse, establish) to progress a metacommunity through one simulation timestep. The order of operations is as follows:

  1. Death : Established individuals in each community experience probabilistic mortality according to species- and habitat-specific mortality rates provided in species. See die.
  2. Birth : Established individuals in each community produce propagules according to species- and habitat-specific birth rates provided in species. See reproduce.
  3. Dispersal : New propagules from each community disperse across the landscape (land) according to species-specific dispersal rates provided in species. The parameter d_kernel specifies the dispersal kernel for new propagules. See disperse.
  4. Movement : Established individuals in each community move from their current cell with species- and habitat-specific movement rates provided in species. The parameter v_kernel specifies the dispersal kernel for previously established individuals. See disperse.
  5. Establishment : Empty spaces in each community are colonized by either a migrant from outside the community with probability imm_rate or by an individual selected at random from the pool of new propagules and moving individuals that arrived in the cell. External migrants are chosen probabilistically from the relative abundances given in gsad. See establish.

See also

run_sim for running multiple timesteps of the simulation.
