Run simulation

Usage

run_sim(steps, metacomm, land, species, gsad, d_kernel = NULL, v_kernel = NULL, imm_rate = NA, save_steps = NULL, report = 0, ID = NA)

Arguments

steps
(required) number of timesteps to run the simulation
metacomm
(required) matrix of lists defining metacommunity (as generated by populate_landscape or returned by this function).
land
(required) matrix or raster of habitat types defining the landscape (as generated by make_landscape)
species
(required) array of species vital rates (as generated by make_species)
gsad
(required) vector defining the global relative abundance of each species. Must be in the same order as species. Defaults to same abundance for all species.
d_kernel
list defining the dispersal kernel of new propagules (see get_dispersal_vec for options). Passed to the form parameter in disperse. Defaults to Gaussian.
v_kernel
list defining the dispersal kernel of previously established individuals (see get_dispersal_vec for options). Passed to the form parameter in disperse. Defaults to Gaussian.
imm_rate
immigration rate. Passed to the parameter m in establish. Defaults to 0.
save_steps
vector of timesteps at which to save the simulation. Defaults to all timesteps, including the initial metacommunity.
report
integer giving the interval at which the function should report its progress to stdout. Defaults to 0 for no reporting.
ID
character string identifying this simulation run when reporting progress

Value

a 3-dimensional array of lists defining the metacommunity at each timepoint in save_steps. Time is in the 3rd dimension.

Description

Runs the simulation on a metacommunity for a fixed number of timesteps.

Details

Given a metacommunity (metacomm, see populate_landscape), landscape (land, see make_landscape), species pool (species, see make_species), and global species abundance distribution (gsad), this function calls run_timestep to progress a metacommunity through a fixed number of timesteps (given by steps). The user may specify which timesteps should be returned using the save_steps parameter. Including 0 in save_steps will save the initial metacommunity.

See also

run_timestep for how the simulation progresses through a single timestep. run_sim_N for running multiple independent simulations on the same set of parameters.
