Aggregate cells

Usage

aggregate_cells(X, Y = NA, dX, dY = NA, form = NA, slide = NULL, locs = NULL)

Arguments

X
(required) x-dimension of the grid or a vector of length 2 giving the dimensions of the grid (x, y)
Y
y-dimension of the grid
dX
(required) number of rows to aggregate or a vector of length 2 giving the dimensions to aggregate
dY
number of columns to aggregate
form
character string indicating how cells should be aggregated (see details).
slide
if form = 'window', a vector of length 1 or 2 indicating the number of cells to slide the window in the x and y directions. Default is one cell in each direction.
locs
if form = 'origin', a matrix of cell locations on which aggregations should be centered. Default is the center cell.

Value

a list of matrices

Description

Groups cells of a grid together and returns a list of cell locations.

Details

This function is used for summarizing simulation output across different spatial grains. The function returns a list of matrices, where each matrix specifies the locations of cells to be aggregated. This list can be passed as the locs parameter in summary functions such as summarize_sim_N and the functions therein. The function implements three methods for aggregating cells:

'partition'
The grid is partitioned so that groups with dimensions (dX,dY) are non-overlapping, starting from the first row, first column. If the number of rows and columns in the grid is not divisible by dX or dY then the last aggregation in each row or column will contain fewer columns or rows of cells than specified.

'window'
A window with dimensions (dX, dY) is slid repeatedly across the grid according the the number of steps given in slide. If slide is less than (dX, dY) this results in overlapping groups of cells.

'origin'
Groups of cells with dimensions (dX,dY) are aggregatedaround a focal cell or set of cells given in locs.
