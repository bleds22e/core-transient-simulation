Abundance profiles

Usage

calc_abun_profile(locs, t_window, sim, N_S)

Arguments

locs
(required) either a two-column matrix of cell locations where species' abundances should be summed or a list of matrices designating multiple spatial units where abundance should be summed (see details)
t_window
(required) either a list containing start and stop specifying that all collected timepoints in that interval should be considered or an explicit vector of timepoints
sim
(required) array of simulation results from a single simulation run (as returned by run_sim)
N_S
(required) number of species in the simulation

Value

an array of species' abundances with dimensions: [timepoint, species, spatial unit]

Description

Calculates species' abundances through time for a simulation.

Details

The function calculates species' abundances over a window of time given in t_window. This can be either a continuous set of timepoints (designated by passing a list with elements start and stop) or an explicit set of timepoints (passed as a vector of integers). Timepoints correspond to actual timesteps in the simulation, even if not all timesteps are saved in sim. Specifying start=1 will return the first timestep, not the first step saved in sim. If the simulation did not save the the first timestep then the function will exit with an error message.

Species abundances are summed over the set of cell locations given in locs. If locs is a list (as generated by aggregate_cells) then each element of the list specifies a matrix of cells (a 'spatial unit') in which abundance should be summed. For example, to calculate species' abundances in each cell separately, pass a list where each element is the coordinates of a single cell. The function returns an array of species' abundances with dimensions: [timepoint, species, spatial unit]. It will include a column labeled '0' for the number of empty spaces.
